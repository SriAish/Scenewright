Screen descriptions

Theme block (applies to every screen)
Light, airy interface: off-white/warm paper background, generous whitespace, a single accent color (deep ink blue or forest green) used sparingly for primary buttons, active states, and mention chips. Dark elements allowed only as accents (buttons, chips, icons), never as large surfaces; the app must read as light mode. Clean sans-serif, clear type hierarchy, subtle borders and soft shadows over heavy dividers. Density is medium: this is a writing and reference tool, so text legibility beats decoration. Statuses are muted color-coded pills (draft gray, running green, completed blue, skipped amber). Entity mention chips are small rounded inline tags with a type icon (person, monster, item).

1. Sign in / Sign up
 Minimal centered card on the paper background: app name, one-line tagline ("plan campaigns, scene by scene"), email/password fields, primary button, toggle between sign in and sign up. No imagery carousel, no marketing sections. Footer link to the SRD attribution page.
2. Campaign dashboard (home)
 Grid or list of campaign cards, each showing title, one-line premise excerpt, status pill, scene count, and last-edited time. Top bar: app name left; right side has a Library link, and the user menu. Above the grid: status filter (All / Draft / Running / Completed) as segmented control, and a prominent "New campaign" primary button. Empty state for new users: friendly illustration-free prompt with two actions, "Start blank campaign" and "Browse adventure directory." Soft-deleted campaigns are not shown.
3. New campaign (modal)
 Small modal over the dashboard: title field, optional premise textarea, status defaulting to Draft, create button. Secondary link at the bottom: "or start from a published adventure," which routes to the adventure directory. Has a second variant when arriving from the directory's "Start campaign from this": title pre-filled with the adventure title, premise stub noting the source, a muted provenance line, and no adventure link.
4. Adventure directory
 Searchable card list of curated free adventures: title, publisher, short description, tag chips (level range, genre), external-link icon opening the source in a new tab. Each card has a "Start campaign from this" button that opens the new-campaign modal pre-filled. A short disclaimer line at the top: the app links to adventures, the GM works from their own copy.
5. Campaign view, shell + Scenes tab (default)
 Persistent campaign header: campaign title (inline editable), status pill (click to change), and actions on the right: "Export PDF" and overflow menu. Below the header, tab bar: Scenes, Characters, Monsters, Items, Notes. The Scenes tab has a view toggle (List / Graph) at its top right and a "New scene" primary button.
 List view: rows with drag handle, scene name, status pill, first line of narration as muted preview text, small icons indicating presence of map/entities, subtle hover state on rows (clickable).
 Empty state: "No scenes yet" with a New scene button and one quiet line reminding that scenes can be written in any order.
6. Campaign graph view (Scenes tab, Graph toggle)
 Full-width canvas (React Flow) with pan/zoom. Scene nodes: rounded cards showing scene name and status color as a left edge stripe; a small badge if the scene has a map. Directed edges with arrowheads and optional small labels ("if players side with the cult"). Toolbar floating top-left: auto-arrange button, zoom controls, "New scene" button. Drawing an edge between nodes creates a link; clicking a node navigates to the scene editor; dragging repositions and persists. Orphan nodes render normally, no warning styling. Canvas background: faint dot grid on the paper background, staying light.
7. Scene editor
 The core writing screen. Header: back to campaign, scene name (inline editable), status pill selector. Main column, in order: Start (labeled rich text section), Narration (large rich text section), End (labeled rich text section). All three are mention-enabled: typing @ opens a compact autocomplete dropdown listing campaign entities with type icons plus a "Create new…" row. Mentions render as inline chips; hovering shows a popover card (name, type, two-line summary, thumbnail if any); clicking opens the entity detail.
 Above the Start section, a collapsible muted panel: "Previous scenes end with…" listing each predecessor scene's End text (plural, since branching allows multiple predecessors).
 Right sidebar, collapsible: Map card (uploaded image thumbnail with replace/remove, plus a "map source link" URL field), then Characters, Monsters, Items sections, each auto-populated from mentions (chips) with an "Add" button for manual additions. Auto-populated chips are visually distinct from manually added ones (e.g. subtle dotted border).
 Autosave indicator, quiet, near the header.
8. Entity tabs (Characters / Monsters / Items, same layout)
 Toolbar: search field, "New [type]" primary button, "Generate" button, and "Import" button. Below: card grid; each card shows portrait/art thumbnail (or type-icon placeholder), name, two-line summary, and a small "appears in N scenes" count from reverse-lookup. Clicking a card opens entity detail. Soft-deleted entities hidden. Empty state includes a hint that @ in any scene creates entities mid-sentence.
9. Entity detail / edit
 Two-column layout. Left: image (upload/replace), name, summary (two lines, used in pickers), type-specific structured fields. For NPCs: Description, Personality Traits, Ability Scores (six-stat compact grid), Relationships, Alignment Tendencies, and a mention-enabled Backstory rich text section. For monsters: statblock fields (CR, type, size, AC, HP, speeds, abilities, actions). For items: rarity, category, attunement toggle, plain-text description (no rich text, no chips).
 Right rail: "Appears in" panel listing scenes, backstories, and notes referencing this entity (each row navigates there); provenance line if forked ("copied from [campaign]") or SRD-derived ("based on SRD: Ghast"); "Save to library" button, only visible when no library entity shares this lineage; soft-delete action in an overflow menu with a note that mentions will show as inactive chips.
10. NPC generation (modal)
 Two-panel. Left, inputs: race, sex, alignment, occupation dropdowns; plot hooks text field; optional freeform steering textarea; Generate button; reserved footnote space for the "generated offline — relationships left blank" note (v1 default state). Right, results: three candidate cards, each with name, description excerpt, personality traits, compact ability score strip, alignment tendency, and a "Use this one" button; "Reroll all" secondary action below. Pre-generation state: right panel shows a quiet prompt line. Picking a candidate opens entity detail in edit mode.
11. Monster / Item finder (modal from the Monsters or Items tab)
 Top: prose search field ("something stealthy that ambushes from water"). Collapsible structured filters: monsters get CR range slider, type, size, environment, alignment; items get rarity, category, attunement. Results: top 3 match cards with name, key-stats line, short description, "Add to campaign" button; "More results" link below. A quiet toggle to browse the full SRD list alphabetically. Footer microcopy: "SRD 5.1/5.2 content, CC-BY" linking to the attribution page. Pre-search state: quiet prompt line, filters collapsed.
12. Import from my campaigns (modal from any entity tab)
 Search field across all the GM's campaigns and the Library. Results grouped by lineage: each group shows a representative header row with a version count, expandable into variant rows; each variant row has a radio button, name, source label (campaign name or "Library"), and two-line summary; "Import copy" primary button. Muted toggle at the bottom: "Show variants from deleted campaigns." If the chosen variant's backstory mentions entities not in the target campaign, a follow-up confirm step: "[Name]'s backstory mentions [X] and [Y]" with two radio options, "Copy referenced entities too (N copies will be created)" and "Flatten references to plain text" (default), Back and Import buttons.
13. Library
 Same chassis as entity tabs but app-level: top bar (app name, Library active, user menu), page title with muted line "Reusable homebrew, independent of any campaign." Toolbar: search, type filter segmented control (All | NPCs | Monsters | Items), "New entry" primary button; no Import or Generate. Mixed-type card grid with a small type icon per card; footers show type-appropriate metadata (monsters: CR; items: rarity; NPCs: optional muted "Saved from [campaign]"). Backstory mentions here autocomplete against library entities only. Empty state: no button in the block itself (entries mostly arrive via "Save to library"), "New entry" remains in the toolbar.
14. Notes tab (inside campaign view)
 Single full-width mention-enabled rich text document titled "Campaign notes," same @ behavior as scene narration. One quiet caption: "Session outcomes, table decisions, anything worth remembering." Autosave indicator.
15. PDF export (small modal)
 Triggered from the campaign header. Shows what will be included (storyline, scenes in reading order, maps, entity lists, entity appendix, notes) as a fixed checklist for transparency, a note that output is black and white, and an Export button with a progress state. No per-section toggles in v1.
16. Attribution / about (static page)
 Plain text page: SRD 5.1/5.2 CC-BY attribution statement, Watabou/tool acknowledgments if used, link back to the app.


Shared components (design once, reuse): mention chip + hover popover, status pill set, entity card, result/candidate card family, modal chassis (header, padding, footer, dimmed backdrop), empty states. Screens 3, 10, 11, 12, 15 are modals; screens 2, 13, and 16 share the app-level top bar; screens 5, 6, 7, 8, 14 share the campaign header + tab bar chassis.
